# 加密货币量化交易平台与自动化 Bot 构建指南

## 一、整体架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        加密货币量化交易平台                                │
├─────────────┬─────────────┬─────────────┬─────────────┬─────────────────┤
│  数据层      │  因子/策略   │  信号/决策   │  执行层     │  账户/风控       │
│  Data        │  Factors    │  Signals    │  Execution  │  Account/Risk    │
├─────────────┼─────────────┼─────────────┼─────────────┼─────────────────┤
│ • K线/深度   │ • 技术指标   │ • 多空/权重  │ • 下单 API  │ • 余额/持仓      │
│ • 订单流     │ • 订单簿因子 │ • 阈值/条件  │ • 撤单/改单 │ • 止损/限仓      │
│ • 实时行情   │ • 回测验证   │ • 频率控制   │ • 模拟/实盘  │ • 日志/审计      │
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────────┘
```

- **数据层**：你已有 `binanceClient` + `kline`（K 线），可扩展深度、逐笔。
- **因子/策略**：在历史或实时数据上算因子，回测/模拟验证后再接入实盘。
- **信号/决策**：根据因子或规则产生买/卖/不操作。
- **执行层**：统一「模拟」与「实盘」两种模式，同一套策略先走模拟再切实盘。
- **账户/风控**：余额、持仓、最大回撤、单笔上限等。

---

## 二、自动化买入卖出 Bot 核心流程

1. **配置**  
   交易对、周期、策略参数、**运行模式（paper / live）**、API Key（实盘或 Testnet）。

2. **数据**  
   定时或事件驱动拉取 K 线/深度（你已有 `fetchCryptoKline`），可选 WebSocket 实时推送。

3. **因子与信号**  
   根据 K 线或订单簿计算因子，再根据规则或简单模型得到：做多、做空、平仓、观望。

4. **执行**  
   - **模拟盘**：不调交易所 API，本地维护虚拟仓位与净值，按「假设成交」更新。
   - **实盘**：调用交易所下单/撤单接口（需签名，见下）。

5. **风控**  
   单笔最大量、总仓位上限、日亏损熔断、最大回撤等，在发单前校验。

6. **日志与监控**  
   每笔信号、每笔下单/成交、账户变动写日志或数据库，便于复盘和调参。

## 三、虚拟盘（模拟盘）—— 用于训练因子与策略

### 1. 交易所官方测试网（推荐先用）

| 交易所   | 测试网 / 模拟盘 | 说明 |
|----------|------------------|------|
| **Binance** | [Binance Testnet](https://testnet.binance.vision/) | 现货 + 期货测试网，送测试 USDT，真实 API 协议，可测下单/撤单/持仓 |
| **Binance Futures Testnet** | https://testnet.binancefuture.com | 合约专用测试网 |
| **OKX** | [OKX 模拟交易](https://www.okx.com/zh-hans/demo-trading) | 模拟盘，需在网页开通 |
| **Bybit** | Testnet | 有测试网环境 |

**用途**：  
- 用**真实行情**（或接近真实）+ **虚拟资金**，验证「因子 → 信号 → 下单」全链路。  
- 不消耗真实资金即可训练/调参，适合做因子生成、阈值和风控的迭代。

### 2. 本地全量模拟（不连交易所下单）

- **回测**：用历史 K 线（如你已有的 `fetchCryptoKline`），按时间顺序逐 bar 算因子、产生信号，按「下一根 K 线开盘/收盘价」假设成交，算净值曲线。  
- **实时模拟**：用实时行情（WebSocket）驱动，同样只维护本地虚拟仓位与 PnL，不向交易所发单。  

这样可以在**同一套因子逻辑**上做：  
- 历史回测 → 检查过拟合、换手、胜率；  
- 实时模拟（或 Testnet）→ 检查延迟、滑点、订单簿影响。

### 3. 建议的「训练因子」工作流

1. **历史数据**：用 Binance 公开 K 线（你已有）或 Testnet 历史数据，做**离线回测**，生成因子、评估 IC、收益、回撤。  
2. **Testnet / 模拟盘**：把同一套因子和策略在 Testnet 或本地模拟里跑一段时间，观察实盘延迟、成交情况、资金曲线。  
3. **小资金实盘**：验证稳定后再用少量真实资金实盘。

## 四、技术实现要点

### 1. Binance 下单（实盘 / Testnet）

- **实盘**：`https://api.binance.com`（你已有），需 API Key + Secret，对请求做 **HMAC-SHA256 签名**。  
- **Testnet**：  
  - 现货：`https://testnet.binance.vision`  
  - 期货：`https://testnet.binancefuture.com`  

同一套「下单/撤单/查余额」逻辑，只需切换 baseURL 和 API Key 即可区分实盘与测试网。

### 2. 统一「模拟 / 实盘」接口

建议抽象一个「执行器」接口，例如：

- `submitOrder(side, symbol, quantity, price?, type): Promise<OrderResult>`
- 实盘实现：调 Binance 的 `POST /api/v3/order` 等。  
- 模拟实现：写入内存/文件的「虚拟订单簿」，按当前价或下一根 K 线开盘价模拟成交，更新虚拟持仓与余额。

这样策略层只依赖「执行器」，不关心是 paper 还是 live。

### 3. 因子与回测

- 因子：用你现有的 `KlineBar[]` 计算（如均线、MACD、布林、订单簿不平衡等）。  
- 回测：按时间遍历 `KlineBar[]`，每根 K 线更新因子 → 信号 → 若为模拟则执行「模拟下单」并更新虚拟仓位。  
- 可先用 CSV/JSON 存历史 K 线，再跑回测，无需实时连接交易所。

## 五、与你当前项目的衔接

- 你已有：  
  - `src/api/binanceClient.ts`：REST 公开接口（多 host 容错）。  
  - `src/api/kline.ts`：`fetchCryptoKline`、`KlineBar` 等。  

可以在此基础上：

1. **加一层「数据服务」**：  
   封装「按 symbol/interval 取 K 线」、可选「写本地缓存/数据库」供回测用。

2. **加「策略/因子」模块**：  
   输入 `KlineBar[]`，输出信号（如多/空/平仓、权重）。  

3. **加「执行器」**：  
   - 模拟实现：维护虚拟仓位与订单列表，按当前价或下一 K 线开盘价成交。  
   - 实盘实现：Binance 签名请求，调 `POST /api/v3/order` 等（需 Node 或后端，避免在前端暴露 Secret）。  

4. **Bot 入口**：  
   定时或事件驱动：拉 K 线 → 算因子 → 得信号 → 调用执行器（模拟或实盘）。  

5. **虚拟盘训练因子**：  
   - 用历史 K 线回测 + 本地模拟盘（或 Testnet）跑同一套因子与策略，不消耗真实资金即可迭代因子和参数。

## 六、安全与合规提醒

- **API Key**：不要放在前端代码或仓库里，用环境变量或后端配置。  
- **实盘**：先用 Testnet 或本地模拟充分验证，再小资金实盘。  
- **风控**：务必做仓位上限、单笔上限、止损/熔断，避免单次错误导致大亏。

## 八、本项目已实现（Bot 平台）

当前仓库已接入：

- **虚拟盘自动交易**：在「Bot 自动交易」标签页配置交易对、K 线周期、MA 参数、运行频率（bar / tick），启动后按策略本地模拟买卖，无需 API Key。
- **较高频支持**：运行频率选「按行情推送 (tick)」时，订阅 Binance WebSocket 实时价格，每 2 秒最多触发一次策略（浏览器环境无法做到交易所级微秒级 HFT，仅能做「按 tick 驱动」的较高频）。
- **大模型因子**：在「设置」中填写 LLM API URL 与 Key（OpenAI 兼容），在 Bot 配置中勾选「启用大模型因子」，策略会将近期 K 线摘要发给 LLM，用返回的 -1~1 分数参与买卖信号。
- **实盘**：实盘下单需在后端用 API Secret 签名，当前前端仅支持虚拟盘；实盘需自建后端或使用 Binance Testnet 代理。
